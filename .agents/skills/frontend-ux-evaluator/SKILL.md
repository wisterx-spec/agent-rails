# Frontend UX Evaluator Skill

## 触发时机
- 完成前端功能开发，进行 UI 自检时
- 产品评审前的 UX 质量预检
- 用户提及"评估 UX"、"检查前端交互"、"Review 页面"

---

## 评估维度

### 1. 交互反馈完整性
- [ ] 所有异步操作是否有 Loading 态
- [ ] 所有操作是否有成功/失败的明确反馈（Toast / Modal）
- [ ] 表单提交是否有防重复提交保护（按钮禁用）
- [ ] 长操作是否有进度指示

### 2. 空态与边界处理
- [ ] 列表为空时是否有 EmptyState 组件（而非裸 div）
- [ ] 数据加载失败时是否有错误态展示和重试入口
- [ ] 搜索/过滤无结果时是否有明确提示

### 3. 组件规范符合性
- [ ] 是否使用了项目统一弹窗组件（无裸 `alert()` / `window.confirm()`）
- [ ] 图标是否来自统一图标库（无文字 Emoji 充当图标）
- [ ] 颜色是否使用语义 token（无 Tailwind 物理色 / 裸 hex）

### 4. 可用性与可访问性
- [ ] 关键操作按钮的可点击区域是否足够（最小 44x44px）
- [ ] 表单字段是否有清晰的 label 和错误提示
- [ ] 危险操作是否有二次确认保护

### 5. 布局与响应式
- [ ] 核心内容区域在不同视口下是否正常展示
- [ ] 长文本是否有截断处理（overflow）
- [ ] 弹窗在小屏下是否可滚动

---

## 输出格式

```
## UX 评估报告：{{页面/组件名}}

### 问题列表

| 严重性 | 维度 | 问题描述 | 建议修复 |
|--------|------|---------|---------|
| 🔴 高  | 交互反馈 | 删除操作无二次确认 | 接入 DeleteConfirmModal |
| 🟡 中  | 空态处理 | 搜索无结果用裸 div | 替换为 EmptyState 组件 |
| 🟢 低  | 组件规范 | 使用了 text emoji ⚙️ | 替换为 lucide-react Settings 图标 |

### 通过项
- [x] 所有异步操作有 Loading 态
- [x] 表单提交有防重复提交保护

### 建议优先处理
1. 🔴 高优先级问题共 X 个，建议在本次 PR 内修复
2. 🟡 中优先级问题共 X 个，可在下次迭代处理
```

---

## 约束

- **NEVER** 自动修改组件代码，只输出评估报告
- 发现问题只能以 `[UX_ISSUE]` 格式记录，由人类决定是否修复
- 不评估业务逻辑正确性，只评估 UX 质量
