# /weekly-report 工作流

> 配置依赖：读取 `project.config.json` → `weekly_report` 字段获取输出目录和前端仓库路径。

### 步骤 1：全栈数据探针检索
依靠现有工具探测并梳理开发流水线数据源：
1. 回顾最近的对话及文件变更，查探当前任务和 Todo 列表情况。
2. 检索并分析当前项目在近 7 天内的 Backend 提交：
   ```bash
   git log --since="7 days ago" --oneline
   ```
3. 如 `project.config.json → weekly_report.frontend_repo_path` 非空，检索前端项目近 7 天提交。

### 步骤 2：输出 A：撰写量化周报正文（Markdown 文件输出）
利用聚合到的有效信息进行提炼，将报告写入文件 `{{OUTPUT_DIR}}/weekly_report_YYYY-MM-DD.md`。

> `{{OUTPUT_DIR}}` 来自 `project.config.json → weekly_report.output_dir`，默认为 `tmp/weekly_reports`。

**【写作语感规范】**
1. **动词要硬**：用"切除""拔除""击穿""强制隔离""阻断"，不用"完善""优化""推进"。
2. **不修饰结果，只陈述事实和代价**：不写"显著提升了稳定性"，写"锁表停机，根因在主库批量指令无切片保护，强制拆分后消除"。
3. **有纵深感**：说清楚问题藏在哪一层（服务端 / 架构底层 / 并发锁 / 模板引擎），不停在表面。
4. **冲突要写出来**：不回避"原有逻辑与新机制打架"、"历史债务阻断迁移路径"这类摩擦。
5. **体量要落地**：不说"大量"，说"43 个用例 / 29 项安全测试 / 6 个归档变更"。
6. **禁止句式**：绝对禁用"持续推进" / "进一步完善" / "有效保障" / "赋能" / 任何形容词加"地" / "显著"。

**【刚性结构大纲】** *(不得遗漏、删改或调换任何章节)*

一、核心工作进展（分为 `功能开发 / 安全质量 / 规范架构` 分组罗列，条目中强制嵌入数字指标）
二、本周亮点（仅列 1~2 条，要求使用一两句话极简事实陈述业务面价值）
三、待优化事项（列出真实存在的漏洞代价、排错卡点或债务摩擦）
四、下周计划（列出 3~5 条，必须遵从标准句式组合：`[硬动词] + [目标层级] + [明确量化指标]`）
五、总结（只用最高 3 句话进行收尾：本周主导目标、完成的用例体量、以及对发布版本节点的事实影响）

### 步骤 3：输出 B：导出防乱码带 BOM 头的任务清单（CSV 格式写入）
必须向目录 `{{OUTPUT_DIR}}` 内生成用于任务跟踪记录的 CSV 数据文件。

**【输出存放控制】**
- **文件命名路径**：`{{OUTPUT_DIR}}/weekly_tasks_{YYYY-MM-DD}.csv`
- **Excel 直读编码**：必须是 **UTF-8 with BOM** 格式。使用 shell 将 `\xEF\xBB\xBF` 魔法头与内容合并写入（必要时可创建父目录）。

**【CSV 数据行列规范】**
- **不可撼动的表头序列**：`专项,专项需求,专项Leader,预计提测时间,预计上线版本或者日期,状态,备注`
- **数值与内容填充规制**：
  - **专项**：分类分组标签（例如 `{{PROJECT_NAME}}`）。同组元素不可跳行，每行必写。
  - **专项需求**：单纯呈现绝对干净的纯文本任务名。**绝对禁止在 CSV 的任何单元格中附带具体的 Git 记录地址、Wiki 链接、工单 URL 等外部引用链接**。
  - **专项Leader**：负责的开发者，缺失则空着。
  - **预计提测时间 / 预计上线版本或者日期**：必须约束在 `YYYY/M/D` 格式（例如 `2026/1/15`）。不确定则留空。
  - **状态（安全枚举）**：只能填充 `[已上线, 待开发, 开发中, 提交测试]` 四个既定词汇。**【必须映射】："下周计划"中的条目也必须作为全新任务行写入 CSV，状态统一标注为 `待开发`。**
  - **备注**：事实代价陈述，无感则留空。

### 步骤 4：最终回复格式（最高优先级绝对限制）
以上两个文件均成功生成并写入后，你在用户对话界面里的**唯一输出**必须严格采用以下格式：

**绝对禁止附加任何其他的问候、总结、过程陈述或废话！**

```
文件已生成：
- 周报：[weekly_report_YYYY-MM-DD.md](file://{{OUTPUT_DIR}}/weekly_report_YYYY-MM-DD.md)
- 进度表：[weekly_tasks_YYYY-MM-DD.csv](file://{{OUTPUT_DIR}}/weekly_tasks_YYYY-MM-DD.csv)
```
